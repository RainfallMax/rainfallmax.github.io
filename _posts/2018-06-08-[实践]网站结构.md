# 关于网站的结构
因为GFW存在所以常年续费vultr的服务器用于SSR，又由于服务器经常被墙，所以会经常换服务器。每次换服务器都会导致线上跑着玩的应用要重新部署一次，所以就学习了docker容器知识。

### 为什么要使用Docker？
1. 更轻松的迁移
2. 一致的运行环境
3. 更轻松的维护和扩展
4. 持续交付和部署
5. 更高效的利用系统资源更快速的启动时间

如何安装参见 docker.com，现在使用的是docker-compose来负责实现对 Docker 容器集群的快速编排，这里要特别注意下Linux需要单独安装，参见https://docs.docker.com/compose/ 。

#### 服务器配置

```
CPU: 1 vCore
RAM: 1024 MB
Storage: 25 GB SSD
```

开始使用的512M内存的主机，但是起不了MySQL服务所以升级到1G内存。

#### 服务器架构
**前端**用 Vue.js+Vue Router+Vuex+axios来构建前端单页应用。
**后端**用Django + REST framework 来写web APIs。
**数据库** 用MySQL 5.7，内存太小，Redis未用（未学）。
**HTTP和反向代理服务器** 采用了Nginx

## 项目结构
```
├── back-end                #后端文件，Django目录
├── front-end               #前端文件，Vue单页应用
├── dbdata                  #数据库文件
├── dbset                   #数据库配置文件
├── SSL                     #HTTPS证书存放
├── .gitignore              #git文件
├── docker-compose.yml      #docker-compose配置文件
└── Nginx.conf              #HTTP和反向代理服务器配置文件
```

### docker-compose.yml	文件

```
version: "3"
services:
  nginx:
    image: nginx:latest
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - ./front-end:/usr/share/nginx/html/
      - ./ssl:/etc/ssl/
    networks:
      - net
    depends_on:
      - db

  web:
    build: ./back-end
    command: python3 manage.py runserver 0.0.0.0:8000
    volumes:
      - ./back-end:/back-end/
    ports:
      - "8000:8000"
    depends_on:
      - db

  db:
    image: mysql:5.7
    volumes:  
      - ./dbdata:/var/lib/mysql
      - ./dbset:/etc/mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: *
      MYSQL_DATABASE: reco
      MYSQL_USER: *
      MYSQL_PASSWORD: *
    ports:
      - "3306:3306"

networks: 
  net:
    driver: bridge
```
